Forking a Repository:

  • Open GitHub Desktop: Launch GitHub Desktop application on your computer.
  • Clone the Repository:
    • Click on the "File" menu and select "Clone Repository...".
    • In the "Clone a Repository" window, you can either select the repository from the list of available repositories or paste the URL of the repository you want to fork into the "Repository URL" field.
    • Click "Clone" to clone the repository to your local machine.
  • Fork the Repository on GitHub:
    • Open your web browser and navigate to the repository on GitHub that you want to fork.
    • Click on the "Fork" button in the top-right corner of the repository page. This will create a copy of the repository under your GitHub account.
  • Add Remote for Forked Repository:
    • In GitHub Desktop, go to the "Repository" menu and choose "Repository Settings...".
    • Click on "Add" under "Remotes" and enter a name for the remote (e.g., origin) and the URL of your forked repository (e.g., https://github.com/your-username/repository).

Making Changes and Committing:

  • Create a Branch:
    • In GitHub Desktop, go to the "Current Branch" dropdown and click on "New Branch...".
    • Enter a name for your branch (e.g., feature/add-new-feature) and click "Create Branch".
  • Make Changes: Use your preferred code editor to make changes to the codebase.
  • Stage Changes:
    • In GitHub Desktop, you will see your changes listed under "Changes" tab.
    • Check the boxes next to the files you want to commit to stage them.
  • Commit Changes:
    • Enter a summary and description of your changes in the fields provided.
    • Click on the "Commit to " button to commit your changes to the branch you created.

Pushing Changes and Creating a Pull Request:

  • Push Changes: After committing your changes, click on the "Push origin" button in GitHub Desktop to push your branch to your forked repository on GitHub.
  • Create Pull Request:
    • In GitHub Desktop, you can directly open the pull request by clicking the "Create Pull Request" button that appears after you push your branch.
    • This will open a browser window where you can review the pull request details, add a title and description, and then click "Create Pull Request" to submit it.
